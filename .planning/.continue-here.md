# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [x] CONSTRAINT: openai SDK ignores http_client headers — Only `default_headers` actually reaches the wire. We wasted time trying to set User-Agent via `httpx.Client(headers=...)`.
- [x] CONSTRAINT: `_get_request_payload` mutations reach the HTTP body — We verified via monkey-patching `client._post` that our payload modifications are serialized into the final JSON request. If the server still rejects it, the issue is the field value/structure, not the injection mechanism.

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Assuming http_client headers propagate | The openai SDK constructs its own header set and ignores most headers on a passed `http_client` | blocking | Always use `default_headers` or `default_query` for provider-required headers |
| Testing payload fixes without wire inspection | It's easy to believe a fix works when `_get_request_payload` shows the field, but the SDK or httpx might still strip it | advisory | Monkey-patch `client._post` or use an HTTP proxy to verify the actual bytes on the wire |
